# Uninformed-Search-Implementation

## Overview

This project implements three uninformed search algorithms to solve the **Milk Jug Problem** with jug capacities **8, 5, and 3 liters**.

The initial state is:

(8,0,0)

The objective is to reach a state where the **8-liter jug contains exactly 4 liters**, while maintaining the invariant that the **total quantity of milk remains 8 liters**.

The implemented search strategies are:

* Depth First Search (DFS)
* Breadth First Search (BFS)
* Iterative Deepening Depth First Search (IDDFS)

Each algorithm reports the solution path along with performance metrics including the number of nodes expanded, number of nodes generated, and total execution time.

---

# Problem Definition

Three jugs are available:

| Jug | Capacity |
| --- | -------- |
| A   | 8        |
| B   | 5        |
| C   | 3        |

The initial state places all milk in the first jug:

(8,0,0)

A valid state must satisfy:

* (0 \leq state[i] \leq capacity[i])
* (A + B + C = 8)

State transitions are generated by pouring milk from one jug into another until either the source jug becomes empty or the destination jug becomes full.

The goal condition is satisfied when:

A = 4

while still maintaining the total milk constraint.

---

# State Representation

Each state represents the amount of milk in the three jugs:

(A,B,C)

To allow efficient storage and comparison, states are encoded as integers:

```
encoded_state = C*100 + B*10 + A
```

This encoding enables efficient use of:

* `unordered_set` for tracking visited states
* `unordered_map` for storing BFS parent relationships

The program provides functions for encoding and decoding states as well as generating all valid successor states from the current configuration.

---

# Performance Metrics

The following metrics are recorded for each algorithm.

### Nodes Expanded

A node is considered **expanded** when successor states are generated from it.
Goal states are detected before successor generation and therefore are not expanded.

### Nodes Generated

Total number of successor states produced during the search.

### Path Length

Number of states in the solution path from the initial state to the goal.

### Execution Time

Measured using `std::chrono` and reported in milliseconds.

Compile the program using a C++ compiler supporting C++11 or later.

```
g++ searches.cpp -o milk_jug_solver
```

# Running

```
./milk_jug_solver
```

The program executes DFS, BFS, and IDDFS sequentially and prints the resulting paths and performance metrics.
